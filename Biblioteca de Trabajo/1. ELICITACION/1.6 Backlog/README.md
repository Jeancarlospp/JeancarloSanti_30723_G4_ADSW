# 1.6 Backlog

Conjunto de requisitos y funcionalidades priorizadas para desarrollo.

## Contenido

- **Product Backlog** - Lista completa de características ordenadas por prioridad
- **Sprint Backlog** - Elementos seleccionados para el sprint actual
- Descripción de cada elemento
- Estimaciones de esfuerzo (story points)
- Prioridad y dependencias
- Estado (Pendiente, En Progreso, Completado)

## Gestión

Se actualiza al final de cada sprint considerando feedback, cambios de prioridad y evolución del proyecto.
