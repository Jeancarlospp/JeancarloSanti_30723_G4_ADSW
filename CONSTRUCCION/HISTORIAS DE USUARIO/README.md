# 1.4 Historias de Usuario

Formato ágil de especificación de requisitos desde la perspectiva del usuario final.

## Contenido

- Identificación de la historia (ID, título)
- Actor/rol del usuario
- Descripción: "Como [rol] quiero [acción] para [beneficio]"
- Criterios de aceptación
- Estimación de esfuerzo
- Prioridad
- Notas y dependencias

## Formato

Cada historia de usuario sigue el formato estándar ágil y facilita la planificación de sprints.
