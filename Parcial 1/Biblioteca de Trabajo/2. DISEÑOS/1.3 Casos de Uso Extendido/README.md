# 1.3 Casos de Uso Extendido

Descripción detallada de los casos de uso del sistema con flujos alternativos y excepciones.

## Contenido

Cada caso de uso extendido contiene:
- Identificador y nombre
- Actores involucrados
- Precondiciones
- Flujo principal paso a paso
- Flujos alternativos
- Flujos de excepción
- Postcondiciones
- Puntos de extensión
- Trazabilidad a requisitos

## Formato

Sigue el formato Cockburn para casos de uso extendidos, permitiendo mayor claridad en la especificación de requisitos funcionales.

## Utilidad

Facilita la comunicación con stakeholders y proporciona base para pruebas de integración.
