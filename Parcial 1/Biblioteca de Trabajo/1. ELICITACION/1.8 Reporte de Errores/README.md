# 1.8 Reporte de Errores

Registro de defectos y problemas encontrados durante el desarrollo y pruebas.

## Contenido

Cada reporte contiene:
- ID del error
- Descripción del problema
- Severidad (Crítica, Alta, Media, Baja)
- Componente afectado
- Paso a reproducir
- Resultado esperado vs. actual
- Asignado a
- Estado (Abierto, Asignado, En Progreso, Resuelto, Cerrado)
- Trazabilidad a requisito

## Ciclo de Vida

Los errores se rastrean desde su reporte hasta su resolución y cierre.
